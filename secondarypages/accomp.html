üéØ CySA+ Exam Strategy

This certification involves critical thinking and practical application. Know your concepts deeply - you won't pass by just memorizing definitions. The exam will ask "What is the BEST solution" or "What is the NEXT step" - pay close attention to these keywords. What may be BEST is not necessarily what is the NEXT step in incident response.

Study Approach: 6-8 hours per weekday, two practice tests daily during exam week, studied for 3 weeks total. The Sybex book provided the best explanations.

Key Focus Areas: Incident Response Life Cycle, Threat Intelligence, Reading Logs, IOCs and Remediations

üìã Incident Response Life Cycle

The cornerstone of CySA+ - when in doubt, follow this lifecycle. It's a lifesaver, especially the Containment step.

1. Preparation

Establish policies, procedures, and tools

2. Detection & Analysis

Identify and investigate incidents

3. Containment, Eradication & Recovery

Isolate, remove threat, restore systems

4. Post-Event Activity

Review and improve processes

üîç Indicators of Compromise (IOCs)

Network IOCs

  • Unusual traffic patterns or spikes in network activity
  • Unauthorized devices or connections on the network
  • DNS or IP address anomalies (traffic to known malicious IPs/domains)
  • Unexpected protocol usage or protocol anomalies
  • Suspicious file transfers or data exfiltration
  • Unauthorized access attempts or login failures

Host IOCs

  • Suspicious files or processes running on a host
  • Unusual system or application behavior (crashes, freezes)
  • Unauthorized software installations or changes to system files
  • System logs showing unusual activity or errors
  • Changes to user account privileges or permissions
  • Evidence of data theft or destruction

Application IOCs

  • Unusual or unauthorized user activity (unusual login patterns, unusual data access)
  • Unauthorized changes to application configuration or settings
  • Known vulnerabilities or exploits associated with the application
  • Suspicious application traffic to known malicious domains or IPs
  • Presence of malicious or unauthorized files within the application
  • Evidence of unauthorized access or data exfiltration within the application

üõ†Ô∏è Nmap Port Scanning Methods

TCP SYN Scan (-sS)

Most popular and fastest scan type. Stealthier than connect scan, works against all functional TCP stacks.

TCP Connect Scan (-sT)

Uses system call, typically for unprivileged users and IPv6 targets where SYN scan doesn't work.

UDP Scan (-sU)

Don't forget UDP ports - they offer plenty of security holes too.

TCP FIN/NULL/Xmas (-sF, -sN, -sX)

Special purpose scans for sneaking past firewalls. Note: Don't work well against Windows.

TCP ACK Scan (-sA)

Maps firewall rulesets. Cannot distinguish open from closed ports.

TCP Idle Scan (-sI)

Stealthiest scan type, exploits trusted IP relationships. Slow and complex.

üî• Firewall Evasion Techniques

-f # Fragment packets (split into smaller pieces)
--mtu <number> # Control packet size (must be multiple of 8)
--scan-delay <time> # Add delay between packets
--badsum # Generate invalid checksum to detect firewall/IDS
-Pn # Skip ping, treat host as alive (bypass ICMP block)
-sn # No port scan, ICMP echo + SYN to 443 + ACK to 80

üí° Key Nmap Concepts

  • Closed port: Server responds with RST flag
  • Filtered port: No response (firewall drops packets)
  • Default scan: SYN with sudo, Connect without sudo
  • -Pn caveat: Bypasses ICMP block but takes longer if host is actually down

‚ö° Hping3 Commands

# Basic TCP connection test
hping3 -S <target IP>

# Ping sweep
hping3 -1 <target IP range>

# Traceroute
hping3 --traceroute <target IP>

# SYN flood attack
hping3 -S -p <port> --flood <target IP>

# UDP flood attack
hping3 -2 -p <port> --flood <target IP>

# OS fingerprinting
hping3 -F -A -V <target IP>

üìä Tcpdump Commands

# Basic packet capture
tcpdump -i <interface> -n

# Filter by protocol
tcpdump -i <interface> -n <protocol>

# Filter by IP address
tcpdump -i <interface> -n host <IP address>

# Filter by port
tcpdump -i <interface> -n port <port number>

# Save to file
tcpdump -i <interface> -n -w <filename>

# Read from file
tcpdump -r <filename>

üéØ CVSS (Common Vulnerability Scoring System)

CVSS provides a standardized method for rating IT vulnerabilities and determining urgency of response. Scores range from 0-10, with 10 being the most severe.

Rating Score Range Description
None 0.0 No vulnerability
Low 0.1 - 3.9 Minor impact, difficult to exploit
Medium 4.0 - 6.9 Moderate impact, somewhat difficult to exploit
High 7.0 - 8.9 Significant impact, relatively easy to exploit
Critical 9.0 - 10.0 Severe impact, very easy to exploit
